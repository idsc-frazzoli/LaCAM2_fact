Determines if the given configuration can be factorized and generates sub-\/instances accordingly. 

This function attempts to partition agents into groups based on a heuristic that evaluates if they can be factorized together. It merges partitions if they meet the heuristic criteria, and if a single partition containing all agents is found, it returns an empty list. Otherwise, it splits the configuration into multiple sub-\/instances based on the partitions and returns the list of these sub-\/instances.

Determines if the current instance is factorable based on a predefined partitioning at a given timestep. 

This function checks if a given timestep exists in the {\ttfamily partitions\+\_\+map} and then filters partitions based on the currently enabled agents. If the filtered partitions contain more than one group, the function invokes {\ttfamily split\+\_\+from\+\_\+file} to create sub-\/instances for each partition.
